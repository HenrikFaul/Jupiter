## Objective

Implement the selected value themes in the existing Android app: explainable safe cleanup, immediate duplicate-review access, truthful storage accounting, a QR-paired expiry-bound Jupiscan Relay, Privacy & Recovery Center, and complete user-facing Jupiscan branding.

## Constraints

1. Work from freshly pulled `main` and push directly to `origin/main` only after quality gates.
2. Preserve applicationId/package, existing Room/pref/file names and public screen routes: those are an update/data contract even though visible product branding is Jupiscan.
3. Do not change exact dedup policy: only full content hash proves exact duplication; Similar photos remain review-only; keeper and Recycle Bin remain mandatory safety boundaries.
4. Do not claim unsupported security. Relay must be labelled trusted-LAN HTTP unless TLS and a verified desktop client are implemented.
5. Do not persist pairing tokens; use secure random entropy, finite expiry, auth before list/download and no cache/referrer leakage.
6. Secret-storage failure is fail-closed; no fallback plaintext or partially persisted remote definition.
7. Update changelog, one version summary, this AI record and the external lessons system when an actual new failure pattern is found.
