## Objective

A létező Jupiter Android file manager design- és funkcionális auditja, majd a supplied `Generatedscreens` referencia-rendszer regressziómentes, valós állapotokra épülő integrálása. A cél nem új app készítése, hanem a meglévő indexelési/deduplikációs/tárhely-felszabadítási funkciók megtartása és a hiányzó UI vagy működés pótlása.

## Authoritative user requirement

A felhasználó magyar kérése: „**teljes mértékben ráhúzd promptok alapján a mellékelt designeokat és képeket a mi kódrepónkra**”, miközben „**funkciókat és direktben kért elnevezéseket semmit nem regresszál**”. A design/prompt-only funkciókat is implementálni kell, az eddig nem designolt működésekhez pedig a guideline szerint UI-t kell adni. Kötelező a `changelog`, a részletes `versioning` rekord, valamint az external Henris lessons rendszer és különösen a FORGE prompt-rendszer használata.

## Implementation constraints

- Additív, regressziómentes fejlesztés: meglévő funkció, route, nyilvános API, enum vagy direkt elnevezés nem távolítható el/írható át önkényesen.
- A valós index az egyetlen autoritatív keresési/dedup forrás; exact dedupot kizárólag teljes hash dönt, perceptuális similar külön marad; v0.49 másodperces identitás- és quick-hash cascade invariánsai nem sérülhetnek.
- A Recycle Bin az egyetlen normál delete út; permanent delete mindig explicit megerősítés; retenció default off.
- `StorageExclusions` minden scan/search/category/analytics/dedup felületen közös policy; szándékos kézi böngészést nem szabad letiltani.
- Sem fake adatok, sem nem működő látszatfunkciók: nem került be dekoratív Similar filter, nem létező App language/PIN/auto-lock, AI-jelölő a cleanupba vagy gyártott keresési snippet.
- Egy fájlt egyszerre egy felelős módosíthat; a párhuzamos munka scope szerint szétválasztott.
- Minden körben kötelező version bump, changelog, versioning summary, AI record és új hibaminta esetén lessons bejegyzés.
- A CI továbbra is a hivatalos build gate; lokális Gradle-eredmény dokumentálható, de zöld CI run ID nélkül zöld CI nem állítható.
