# v0.19.0 — App-wide video thumbnails, Compress polish, Duplicate-cleanup merge

**Date:** 2026-07-02
**Change package:** 260702_24
**Version:** 0.19.0
**Scope:** Jupiter Android app development (not managed device files).

## Goal / why

User feedback from on-device screenshots:
1. **Compress** showed only file sizes — no filename, no preview thumbnail; and no estimated result size
   before compressing. Also "show previews everywhere" — video thumbnails were missing app-wide.
2. **Duplicates and Smart Merge are the same feature** in two menu items — merge into one
   **"Duplicate cleanup"** combining both capabilities.

## Root cause (thumbnails)

Video thumbnails fell back to a clapperboard icon everywhere because **Coil had no `VideoFrameDecoder`
registered** (coil-video was a dependency but its decoder must be added to the `ImageLoader`).

## Added / Modified

- **`JupiterApp.kt`** — now implements `coil.ImageLoaderFactory`; `newImageLoader()` registers
  `VideoFrameDecoder.Factory()`, so `AsyncImage` renders real **video-frame thumbnails app-wide**
  (Compress, duplicate rows, category browser, dual-pane…). Kept `@HiltAndroidApp` +
  `Configuration.Provider` + the download-index observer.
- **`data/media/MediaCompressor.kt`** — `estimateCompressedSize(item, preset, dims)`: pure off-main
  heuristic (video: `bitrate/8 × duration` + audio; image: `targetPixels × ~0.28B`), clamped to the
  original size; never throws.
- **`feature/compress/CompressViewModel.kt` + `CompressUiState.kt`** — compute a per-preset estimate map
  when a source is picked.
- **`feature/compress/CompressScreen.kt`** — picker rows show the **full filename** + a working
  thumbnail; each Target-quality preset shows **"≈ N MB"**; detail preview now renders video frames.
- **`feature/cleanup/DuplicatesScreen.kt` + `DuplicatesViewModel.kt`** — retitled **"Duplicate
  cleanup"**; `selectDuplicatesKeepingBest()` one-tap-selects every non-best copy per group (max
  `MediaQuality.score`, then size, then lastModified) — Smart Merge's capability — alongside the
  existing manual multi-select + delete-to-trash. Duplicate rows now show real **thumbnails** (Coil,
  icon fallback).

## Removed (merge)

- `feature/cleanup/SmartMergeScreen.kt`, `SmartMergeViewModel.kt`, `SmartMergeUiState.kt` deleted;
  `Destination.SmartMerge` + its `JupiterNavHost` route + the "Smart Merge" More tile removed. The
  "Duplicates" tile is retitled "Duplicate cleanup".

## Lessons captured this iteration

- **Coil needs `VideoFrameDecoder` explicitly registered** (via an app-wide `ImageLoaderFactory`) to
  render video thumbnails — coil-video on the classpath is not enough. Fix once, globally.
- **When two screens look identical to the user, merge them** — absorb the unique action (keep-best)
  into the survivor and delete the rest, greppng for dangling references. (Sync to
  `…/lessons-system/lessons_orchestrator.md`.)
