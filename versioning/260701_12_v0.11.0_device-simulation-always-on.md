# v0.11.0 — device simulation always-on (auto-boot emulator) + robust screencap

**Date:** 2026-07-01
**Change package:** 260701_12
**Version:** 0.11.0
**Scope:** MONOLITH APK Analyzer platform development (not analysed apps).

## Goal / why

User: a static-only run (no device, no screens) is unacceptable — "ki kell kényszeríteni mindig hogy teljesen
elemezzen, device szimulációkat végrehajtson"; the whole benchmark depends on analysing SCREENS (the developer-spec
describes screens the app must reach). So MONOLITH must ALWAYS run the app on a device, auto-provisioning an emulator
when none is attached. Install whatever software is needed and record it in dependencies.md. No regression.

## Investigation

The machine already had the emulator binary, an `android-34` system image, and two AVDs (`Pixel_8_Pro`,
`forge_test`) — nothing was booted at analysis time, so the dynamic layer skipped. Validated that `forge_test` boots
headless to **Android 14 / x86_64 in ~1 min** and that `adb exec-out screencap -p` yields a valid PNG (the plain
`shell screencap -p <file>` form prints "usage" on this Android-14 image — the latent capture bug).

## Added

- **`layers/emulator.py`** — `ensure_device(ctx)`: returns an attached device if present, else **auto-boots a
  headless emulator** and waits for `sys.boot_completed`. It picks a configured/first AVD, and if none exists tries
  to **create one** (`avdmanager create avd` from an installed system image). Launches the emulator detached
  (`-no-window -no-snapshot -no-boot-anim -gpu swiftshader_indirect …`), polls up to `boot_timeout` (240 s),
  dismisses the keyguard, and returns the serial + whether we started it. Bounded + honest: returns `None` + a reason
  if no emulator/AVD/image is available or boot times out (the dynamic layer then records an honest SKIP — never
  hangs). Plus `stop_emulator()` (respects `keep_emulator`) and SDK helpers (`_list_avds`, `_installed_system_image`,
  `_find_tool`).

## Fixed

- **`layers/dynamic_apk.py` `_screencap`** — now uses **`adb exec-out screencap -p`** (binary bytes straight to the
  host, robust across Android versions) with the on-device file+pull as a fallback. This is the real fix for the
  "no screens captured" bug on modern emulators (v0.10.0 fixed the honest empty-state + capture retry; v0.11.0
  guarantees a device is present AND that capture actually works).
- **`layers/dynamic_apk.py` `run()`** — the no-device path now calls `ensure_device()` (device simulation) before
  giving up; records the emulator provenance on `dynamic.emulator` (autoStarted, avd, source).

## Modified

- `config.py` — `dynamic.auto_emulator` (True), `avd`, `emulator_headless` (True), `keep_emulator` (True),
  `boot_timeout` (240). `dependencies.md` — a "Device simulation is MANDATORY (auto-provisioned)" section with the
  fresh-machine `sdkmanager`/`avdmanager` setup commands. Version → **0.11.0**.

## Acceptance criteria (met)

- With no device attached, MONOLITH auto-boots a headless emulator and runs the full dynamic layer → screens are
  captured and analysed (validated: emulator boots, `ensure_device` returns the serial, `_screencap` yields a valid
  1.2 MB PNG). Only when an emulator genuinely cannot be provisioned does it SKIP honestly. ✔

## Regression (verified)

Engine + dynamic + emulator import clean; the emulator boot is bounded/guarded (can only add time, never break a
run); all prior invariants (26 sections, 17-dim scorecard, 1000 QA, 100 corroborators, 163 benchmark checks) hold.

## Risk / rollback

Medium (booting an emulator is heavier + environment-dependent). Mitigated: fully guarded + honest SKIP on failure;
`dynamic.auto_emulator=false` disables it. Rollback = restore the old no-device SKIP in `dynamic_apk.run()`.
