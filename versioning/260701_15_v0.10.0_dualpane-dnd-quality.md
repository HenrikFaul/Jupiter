## Goal / why

The user reported that Dual Pane, though present, still had "no use": the specific
missing capability was **long-press-and-drag a file to drop it into the other pane's
folder** (copy/move), plus a toggle bar and a richer feature set "like a dual-pane Total

scoring/comparison** was computed but not made visibly actionable — the user wanted the

## Added / Modified

- **`feature/browser/DualPaneScreen.kt` (modified)**
  - **Drag-and-drop transfer** via stable pointer gestures (`detectDragGesturesAfterLongPress`
    + `onGloballyPositioned` window-coordinate geometry), NOT the version-fragile
    `androidx.compose.foundation.draganddrop` API (compile-safety without local compilation).
    Long-press lifts the item (or the whole selection when the source pane is in selection
    mode); a floating ghost follows the pointer; on release the drop position is hit-tested
    against the other pane's captured folder-row / pane bounds → destination = that folder,
    else the other pane's `currentPath`; same-pane release is a no-op.
  - **Drop-mode toggle** (Copy | Move, default Copy) governing whether a drop calls
    `copySelectedTo` or `moveSelectedTo`; the explicit action-bar Copy/Move buttons remain.
  - **Compare** toolbar toggle: tints rows whose name also exists in the other pane
    (in-memory name match), for at-a-glance same-file spotting. Kept Swap/Equalize,
    active-pane accent, `OperationProgressCard`, dense Coil thumbnails; destination pane
    `refresh()`es after a completed transfer.
- **`feature/cleanup/DuplicatesScreen.kt` (modified)** — each `DuplicateGroup` sorted
  best-first by `MediaQuality.score` DESC then `sizeBytes` DESC; the top copy badged **BEST**,
  the rest presented as removable; per-file size + quality label + a relative indicator
  ("higher resolution"/"same quality"). Selection, delete, tap-to-open and copy-path unchanged.
- **`feature/cleanup/SmartMergeScreen.kt` (modified)** — each recommendation rendered as
  **KEEP · best quality** (primary pill) vs **REMOVE** (muted), with the keep reason made
  explicit (strictly-highest probed score, else largest/newest). Keep/remove/delete unchanged.
