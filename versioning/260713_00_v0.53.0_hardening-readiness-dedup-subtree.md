# Jupiter v0.53.0 — hardening: readiness, dedup idempotency, subtree index coverage

Date: 2026-07-13  
Branch: `main`  
Scope: data/index, data/file-ops, data/trash, core/path-policy, docs

## User requirement source

The user asked to continue from the MD audit gaps without functional regression:

- MediaStore delta is not yet true generation/version metadata delta.
- Subtree operations lack a full mutation gateway + operation journal/saga.
- Room still had `fallbackToDestructiveMigration()`.
- Readiness model was not a shared coverage model.
- 40k image benchmark / device instrumentation evidence was missing.
- Arrival notification was not persistently idempotent.
- `.Trash` exclusion was a utility, not a DI-provided PathPolicy.
- Background checkpoint/progress was not yet full Definition-of-Done.

## Implemented in this round

### Room / migration

- Bumped `FileIndexDatabase` to v6.
- Added explicit v5→v6 migration.
- Removed general `fallbackToDestructiveMigration()` from the index database builder.
- Added `dedup_decision` table:
  - `decisionKey` primary key
  - `kind`
  - `newPath`
  - newline-separated `existingPaths`
  - `algorithmVersion`
  - `createdAt`
- Extended `index_state`:
  - `checkpointJson`
  - `mediaStoreVersion`
  - `lastMediaStoreGeneration`
  - `lastDeltaSyncAt`

### Dedup arrival idempotency

- `DuplicateDetector` now persists a canonical SHA-256 decision key before emitting an alert.
- The key is built from:
  - decision kind
  - algorithm version
  - sorted unordered set of new + existing paths
- Replays/observer bursts/restarts produce no second alert for the same canonical decision.
- Added regression test:
  - `DuplicateDetectorTest.sameArrivalDecisionIsPersistentlySuppressedOnSecondPass`

### Delta marker state

- `IndexStateRepository.recordDeltaSync(version, generation)` records the last reconciler marker.
- `DedupReconciler` records the baseline marker and every successful checkpoint advance into Room `index_state`.

### Readiness

- Added `StorageReadiness`, `Coverage`, `PipelineStatus`.
- Added `IndexReadinessRepository` and Room-backed `IndexReadinessRepositoryImpl`.
- Coverage dimensions:
  - metadata
  - exact hashes
  - image descriptors
  - structural descriptors

### PathPolicy

- Added DI seam:
  - `PathPolicy`
  - `PathClass`
  - `DefaultPathPolicy`
- `StorageExclusions` now delegates to `DefaultPathPolicy`.
- `StorageAnalyticsRepositoryImpl` consumes the injected policy.

### Subtree copy/restore index coverage

- Successful COPY now indexes the full created target tree, not only the top-level target.
- Successful Recycle Bin restore now indexes the full restored tree, not only the root.
- MOVE/rename keeps the previous atomic Room subtree rewrite.

## Still explicitly not claimed complete

- Full MediaStore generation/version metadata delta with delete/rename/move reconciliation is still not complete.
- Full operation journal/saga crash recovery is still not implemented for every file operation.
- 40k image corpus benchmark, reboot/process-kill instrumentation, and OEM/device acceptance remain device-bound gates.
- The readiness repository exists and is wired, but migrating every UI consumer to the new read model is a follow-up.

## Regression guard notes

- Do not reintroduce row-count completeness checks.
- Do not remove the v5→v6 migration; the dedup decision table is now part of notification idempotency.
- Do not index only copy/restore roots; subtree descendants must be upserted immediately.
