# AI development record — v0.12.0

## Objective

Make the install step of the dynamic layer robust and self-healing: diagnose the specific reason an APK fails to
install, uninstall-first, auto-retry on an ABI-compatible emulator, and always give a concrete remediation — so the
benchmark reliably reaches (and screenshots) the app instead of reporting a generic "failed to install".

## Authoritative user requirement

The user (with an accurate technical brief) required: treat install failures first as ABI/signing/minSdk/emulator
mismatch; always check the emulator ABI; inventory the APK's native libs; fall back to another system image; mark
install/launch/permission/runtime failures separately in the report; never answer just "AI error"/"unsupported";
always give a concrete remedial step. Install whatever software is needed.

## Process (per DEVELOPMENT_CONTROL_PROMPT + CodeGraph-first)

Diagnosed graph-first: read the exact install code path (`dynamic_apk.run` install block) + the emulator's real
abilist (`x86_64,arm64-v8a` via ndk-translation) before editing. Built additively; then updated changelog +
versioning + lessons + dependencies + version bump; re-verified.

## Key decisions

- The emulator already translates arm64 (abilist includes arm64-v8a), so the failing class is armeabi-v7a-only apps
  and signature/minSdk — handle each specifically.
- Uninstall-first is the cheapest, highest-value fix (signature mismatch), applied unconditionally.
- ABI fallback selects a system-image ABI that *covers* the app ABI (x86_64 image runs arm64; x86 image runs
  armeabi-v7a), installs it via sdkmanager if allowed, creates a `monolith_<abi>` AVD, boots, retries — bounded +
  honest, gated by `allow_image_download`.
- Nothing new needed installing on this machine (image + AVDs present); the fresh-machine commands are documented.

## Verification

Unit-tested `_diagnose_install` (5 classes) + `_image_abi_for` (arm64→x86_64, armeabi-v7a→x86). On the live
emulator: `ensure_device(required_abis)` returns the compatible device, uninstall-first + install → Success,
screencap → valid 1.2 MB PNG. Invariants intact.

## Lessons captured this iteration

Extended the Android platform lesson (`[ADB-SCREENCAP-EXEC-OUT + AUTO-EMULATOR]`) with the INSTALL side:
install-failure must be classified specifically (NO_MATCHING_ABIS / OLDER_SDK / signature / parse / storage) with a
concrete fix; uninstall-first; compare APK `lib/<abi>` against the device `ro.product.cpu.abilist` (which includes
TRANSLATED ABIs — x86_64 images run arm64); fall back to an ABI-matched image; never a generic "failed".
