## Fixes / changes

- **(1) App-storage grant auto-start** (`AppStorageViewModel.onResume`): the `AppOpsManager` grant can
  lag the return from Settings, so a single on-resume `load()` saw "no access" and stayed on the
  prompt. `onResume()` now polls `hasUsageAccess()` every 250 ms for ~3 s and loads the moment access
  appears (or after the grace window), so the per-app scan starts on its own with no re-entry. When
  access is already held (e.g. returning from the uninstall dialog) it loads immediately.
- **(4) Large-files trash exclusion** (`FileIndexRepositoryImpl.largeFiles`/`allFiles`): the Large
  files list is index-backed and still contained stale trash rows. It now filters (and prunes) any
  `StorageExclusions`-excluded path, so a trashed file never appears there and can't open to
  "Not found". Complements v0.43.0 (which covered category/album/search/analytics).
- **(2) Duplicates size filter + Keep-best removal** (`feature/cleanup`): removed the "Keep best"
  toolbar action and banner (the âś“âś“ toggle covers select-all/deselect-all). Added a `SizeFilter`

## Remaining risk / rollback

- Low. The grant poll is bounded (~3 s) and cancels with the screen; the Large-files filter reuses the
  tested exclusion; the size filter is a pure view filter over the existing groups (selection/delete

  is left as harmless dead code. Rollback = revert.

## Lessons captured this iteration

- **A permission grant is not instantly observable on the return frame.** Relying on a single
  on-resume check races the OS propagating the AppOps grant; a short bounded poll is what makes
  "granted â†’ scanning" feel immediate without a manual re-entry.
- **Read-time exclusion must cover every read, not just most.** v0.43.0 filtered category/album/search
  but not the index-backed Large-files list; the same "Not found" resurfaced there. Routing every
  list read through the one exclusion (and pruning on sight) closes the last surface.
