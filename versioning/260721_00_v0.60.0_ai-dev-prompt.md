# AI development record — Jupiscan v0.60.0 image-descriptor lifecycle

## Objective

Continue the existing Android application by making persisted image metadata more reliable for
indexing and duplicate review, commit/push directly to synchronized main and deploy through the
existing GitHub Actions release. Functional regression is forbidden.

## Mandatory interpretation

- This is the existing Kotlin/Compose/Room Jupiscan application, not a new Python or desktop app.
- The attached VISOR document is algorithmic roadmap input. Do not create a second Python feature
  database, PyQt/Tauri plugin, Torch/FAISS runtime or claim the full nine-layer system.
- Precision takes priority over broad Similar recall because the result appears in cleanup UI.
- Exact deletion evidence remains full content hash; visual evidence is review-only.
- Persist only bounded metadata with an explicit producer version and source identity lifecycle.
- Preserve owner files `Requirements/` and `versioning.zip`.

## Required execution

1. Fast-forward `origin/main`, remain on main and inspect chronological versioning/changelog,
   README, lessons and current image producer/DAO/repository/backfill/grouping/cache paths.
2. Audit all Room versions and reject destructive fallback.
3. Add a minimal image descriptor version, additive migration and version-aware ready/backlog/query
   predicates without a second table.
4. Preserve the version on unchanged byte+type identity and clear it with every derived descriptor
   on content/type change.
5. Make production comparison return explainable three-family evidence and close the unbounded
   third-family false-positive path.
6. Guarantee LSH candidate coverage at the inclusive public threshold while retaining stacked
   verification and complete-link grouping.
7. Add deterministic keyset backfill so retryable early rows cannot starve the library; count every
   visited row, recheck the authoritative backlog and batch each decoded page in one Room transaction.
8. Make readiness observe descriptor writes directly, treat mixed sentinels as incomplete and
   invalidate previous image-decision cache.
9. Reconstruct missing oldest Room migrations, share one production/test chain and export the v11
   schema.
10. Add positive, hard-negative, boundary, lifecycle, readiness, cache and real migration tests.
11. Run targeted and full tests, lint, debug/release APK builds and retained-data API 34 upgrade.
12. Update What's New, README, changelog, detailed versioning records and external lessons.
13. Commit/push directly to main; wait for the concrete Android CI run and build-latest assets; then
   append run/hash evidence in a documentation-only follow-up commit.

## Acceptance gates

- A complete current image stack is queryable; a stale/missing-family/missing-geometry stack is not.
- Terminal current UNHASHABLE rows do not retry forever and never match.
- A third hash family at distance 64 cannot be hidden by two equal families.
- Existing PNG/JPEG+resize positive still passes the production decision.
- A low-contrast JPEG q50 re-encode can use only the bounded d≤10, p≤4, a≤4, score≤6 recovery path.
- Exactly eight dHash bit differences distributed across the former byte bands are considered.
- An entirely retryable first 100-row page does not block row 101 onward.
- 205 backlog rows persist as 100+100+5 page transactions; a final recount catches cursor-behind rows.
- Ready + missing equals image total across valid, terminal, mixed-sentinel, missing-geometry and
  obsolete-version states.
- Descriptor coverage emits after a targeted update without changing `index_state`.
- v1 opens through v11 under Room validation and v10→v11 preserves unrelated evidence.
- Previous image-decision cache is rejected.
- Full unit suite, lint task, debug/release assemble, emulator migration/open and whitespace checks
  are green before push.
- CI/deployment is claimed only from a concrete successful GitHub Actions run.

## Prohibited regressions

- Do not relax exact hash identity or make Similar destructive.
- Do not remove complete-link verification, keeper protection, filters, Select all/Deselect all or
  Recycle Bin workflow.
- Do not reset content hashes, media descriptors, outbox/checkpoint state or application identity.
- Do not bypass Room migration failures with destructive recreation.
- Do not claim D4/crop/semantic/40k-device performance without implementing and measuring them.

## Completion state

Local implementation and verification are recorded in the paired v0.60.0 summary. Remote commit,
CI run and release asset evidence are added only after they actually exist.
