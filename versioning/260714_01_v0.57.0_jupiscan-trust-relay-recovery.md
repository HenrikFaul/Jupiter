## Goal and authoritative requirement

The owner selected five product themes: “biztonságosan döntő tárhely-asszisztens”, Explainable Cleanup, Instant Download Guard, Storage Truth, QR-paired time-limited Relay and Privacy & Recovery Center. They also required: “a JUPITER nevet felejtsd el, jupiscan a neve” and “REGRESSZIÓ NÉLKÜL”.

This package turns the research themes into additive, existing-surface features. It does **not** rename the Android package, Room databases or preference keys because doing so would break upgrade identity/data continuity—the visible brand is Jupiscan.

## Acceptance criteria

1. Installed launcher app label is `Jupiscan`; no current user-visible “Jupiter” label remains in the touched surfaces.
2. The Duplicate hero says both what was found and how it is safe; no selection action can select the protected keeper.
3. A duplicate notification tap waits for normal onboarding/permission flow and then opens `Destination.Duplicates`; it does not create a separate destructive action.
4. Storage total/free are platform values and the analysed category total is visibly distinct.
5. Relay has no stable listening port or persisted pairing secret; unauthorized/expired requests are rejected; internal index file links preserve the paired session token.
6. Privacy hub links route to existing working destinations rather than mock data.
7. Existing update identity/data names remain unchanged and the app compiles/tests pass.

## Non-regression contract

- Exact-vs-similar policy, size filters, sort, Select all/Deselect all, keeper protection and Recycle Bin are retained.
- Arrival detection still first stores the canonical decision before delivery and retries notification when Android previously blocked it.
- Remote secret storage fails closed rather than returning to plaintext; a failed save does not add a connection row.
- Relay does not start silently; it serves only direct regular files under the existing Downloads root, rejects traversal and stops on expiry/ViewModel destruction.
- No destructive Room migration or package/applicationId rename was introduced.

## Remaining risk and rollback

- Relay is deliberate short-lived bearer-token access over a trusted local network. Browser-visible HTTP URLs are not TLS/E2E encryption. A cross-network, two-way companion transfer requires separate protocol, certificate and threat-model work.
- Android can report complete volume capacity but not file-level category ownership for protected system/app data. The UI makes the gap explicit.
- Production signing needs owner-controlled release-key/CI secret provisioning. This record does not claim that debug-signing is production-ready.
- Rollback is a normal git revert. Existing remote download directory is read first when present; no user file migration occurs.

## Lessons captured

- A token in a root pairing query must be propagated to relative file links (or replaced with an authenticated cookie/header); otherwise a paired directory page still yields unauthorised downloads. Added as a regression test/review invariant for local file servers.
