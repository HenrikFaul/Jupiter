## Added this iteration (all under `data/index/dedup/`)

- **`SimilarityScorer` + `SimilarityModel`** ‚Äî the fusion engine (blueprint ¬ß4.7 / ¬ß9 / ¬ß10 /
  similarity-score-model). Combines the six `SimilarityLayer`s into a [0,100] score ‚Üí one of six
  `DedupTier`s (UNRELATED ‚Üí WEAK ‚Üí POSSIBLE ‚Üí PROBABLE ‚Üí VERY_LIKELY ‚Üí EXACT) + confidence +
  human `explanation` + per-layer `contributions`. Rules: exact identity always dominates to EXACT;
  per-`TypeClass` weight vectors (image‚Üíperceptual, APK‚Üístructural/signer, text/code‚Üísemantic,
  binary‚Üísample), each summing to 100; hard `Veto`s (signer/size/duration/page-count mismatch) CAP
  the score to `REVIEW_CAP` (59), never raise it.
- **`TextSimHash`** ‚Äî 64-bit SimHash for text/code near-duplicates (the semantic layer for those
  types): formatting-insensitive tokenization + FNV-1a token hashing; Hamming distance ‚â§ 3 = near.
  Survives reformatting/whitespace/case/small edits.
- **`ApkIdentity` + `ApkComparator`** ‚Äî APK structural layer: package + signer + versionCode ‚Üí
  SAME_EXACT / SAME_APP_UPDATE (versioned family, not a delete) / DIFFERENT_SIGNER (repackaged ‚Üí
  vetoed, never "the same app") / UNRELATED, mapped onto a fusion signal + veto.
- **`SampleOffsets`** ‚Äî deterministic size-seeded sample byte-ranges (first/last/middle + interior)
  for the chunk/sample prefilter layer for large binaries.
- **Explainable alerts** ‚Äî `DuplicateAlert` gains `tier` + `explanation`; `DuplicateDetector` fills
  them via the scorer (EXACT identity ‚Üí EXACT tier; image near-dHash ‚Üí perceptual-layer tier +
  "same picture, different size or format").

## Verification (real, in CI)

- `SimilarityScorerTest` ‚Äî exact dominance; strong image perceptual reaches a duplicate tier; the
  SAME perceptual score weighs less for a binary (0) than an image; a veto caps to REVIEW; empty
  signals ‚Üí UNRELATED; tier-boundary mapping; every type's weights sum to 100.
- `TextSimHashTest` ‚Äî identical text distance 0; reformat/case stays near; unrelated far; empty
  never matches; similarity ordering.
- `ApkIdentityTest` ‚Äî the four relations + missing-signer handling + a signer mismatch capping the
  fused score.
- `SampleOffsetsTest` ‚Äî small-file whole-cover; determinism; ascending/in-bounds/non-overlapping;
  head+tail coverage.
- All pure JVM (fast, deterministic) except the unchanged detector tests. CI to be confirmed green.
