# v0.11.0 — Room-backed file index (fast search + indexer foundation)

**Date:** 2026-07-01
**Change package:** 260701_16
**Version:** 0.11.0
**Scope:** Jupiter Android app development (not managed device files).

## Goal / why

The user asked for indexing so the app is faster at reading files and generally:

> "jó lenne egy indexálást is bevezetni az alkalmazásba hogy indexálja a fájlokat hogy
> gyorsabb legyen az app használata a fájlok beolvasásához és úgy általában!"

This introduces a persistent Room index of file metadata, a background indexer, and an
instant-search fast path — the foundation the later duplicate-hash reuse and near-dup
features build on.

## Added

- **`data/index/FileIndexEntry.kt`** — `@Entity(tableName="file_index")`, PK `path`, indices on
  `parentPath` + `name`; columns: parentPath, name, sizeBytes, lastModified, typeName, isDirectory,
  extension, contentHash?, indexedAt.
- **`data/index/FileIndexDao.kt`** — `@Dao`: `@Upsert upsertAll`, `childrenOf(parent):Flow`,
  `searchByName(pattern LIMIT 500):Flow`, `getByPath`, `hashIfUnchanged(path,size,mtime)`,
  `deleteByParent`, `deleteStale(parent, keepPaths)`, `clear`, `count():Flow`, `maxIndexedAt`.
- **`data/index/FileIndexDatabase.kt`** — `@Database(version=1, exportSchema=false)`.
- **`domain/repository/FileIndexRepository.kt` + `data/index/FileIndexRepositoryImpl.kt`** —
  `observeChildren`, `search`, `replaceChildren` (upsert + prune stale), `upsert`,
  `hashForUnchanged`, `putHash`, `clear`, `stats():Flow<IndexStats>`; entry↔FileItem mapping.
- **`domain/model/IndexStats.kt`** — `IndexStats(indexedCount, lastIndexedAt)`.
- **`di/IndexModule.kt`** — `@Provides` the Room database (`jupiter_index.db`,
  `fallbackToDestructiveMigration`) + DAO; a separate abstract `@Binds` module wires
  `FileIndexRepository` → `FileIndexRepositoryImpl`.
- **`data/index/IndexingWorker.kt`** — `@HiltWorker`/`@AssistedInject` `CoroutineWorker`: if no
  All-Files-Access → success no-op; else walks storage in batches (upsert), **excluding**
  `Android/data`, `Android/obb`, `.thumbnails`, `.trashed`; cancellable + crash-safe.
- **`data/index/IndexingScheduler.kt`** — enqueues the worker as a unique `OneTimeWorkRequest`
  ("file-index-rebuild") and exposes its status.

## Modified

- **`data/preferences/SettingsDataStore.kt`** — `+ indexingEnabled: Flow<Boolean>` (default true) + setter.
- **`feature/search/SearchViewModel.kt`** — injects `FileIndexRepository` + `SettingsDataStore`; when
  indexing is enabled it renders cached index matches **instantly**, then reconciles with the
  authoritative live walk (index empty/disabled ⇒ original walk-only behavior; walk failure keeps
  the instant results). Existing natural-language/AI path preserved.
- **`feature/settings/*`** — new **Indexing** section: enable switch, "Rebuild now", and an
  "N files indexed" status; existing controls + `SettingsScreen(onOpenRoute,onBack)` signature intact.
- **`app/build.gradle.kts` + `gradle/libs.versions.toml`** — Room runtime/ktx + `ksp(room-compiler)`;
  `versionName` → 0.11.0.
