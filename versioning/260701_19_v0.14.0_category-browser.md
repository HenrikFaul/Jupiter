# v0.14.0 — Instant MediaStore category browser + research artifacts

**Date:** 2026-07-01
**Change package:** 260701_19
**Version:** 0.14.0
**Scope:** Jupiter Android app development (not managed device files).

## Goal / why

Two user asks in one round:
1. A **20-discipline research sweep** (≥1000 sources) → **100 cross-method verification factors** that
   re-check *existing* capabilities with new methods, plus process the submitted Google-Play UX
   preference research and build only the genuine gaps toward an "ultra-premium" bar.
2. The flagship UX gap that research surfaced: users' #1 loved feature is **one-tap, fast category
   browsing** (images/videos/music/docs). Jupiter showed category *usage* but had **no device-wide
   media listing** — tapping a category opened Analytics.

## Added

- **`docs/RESEARCH_100_VERIFICATION_FACTORS.md`** — 100 factors (10 each across Security/MASVS,
  Usability, Data-integrity, Performance, Accessibility/WCAG, Play-quality, Duplicate-detection,
  Search/indexing, Remote-reliability, Privacy/GDPR), each re-verifying an implemented capability via a
  different method + standard + pass criterion + automation; 629-source appendix. Round-2 backlog: the
  10 disciplines cut off by provider session limits.
- **`docs/UX_PREFERENCES_GAP.md`** — the Play preference research mapped to the code; genuine gaps
  identified (category browser, permission/privacy trust surface, one-tap-action audit); present
  features explicitly not rebuilt.
- **`data/media/MediaStoreCategorySource.kt`** — per-category MediaStore queries (Images/Video/Audio
  collections; Files+mime/extension for documents/archives/apks; Downloads by relative path), cursor-
  safe (`use{}`), off-main (`@IoDispatcher`), never crashes (returns empty on failure/missing
  permission), `CategorySort` (Date/Name/Size). Maps rows → `FileItem`.
- **`feature/categories/{CategoryBrowseScreen,CategoryBrowseViewModel,CategoryBrowseUiState}.kt`** —
  grid + Coil thumbnails for images/videos, dense list otherwise, sort menu, count + total size header,
  honest loading/empty/error; VM reads the category from `SavedStateHandle` (unknown → safe fallback).

## Lessons captured this iteration

- **Salvage partial workflow output from `journal.jsonl`** when a synthesis agent dies on a rate limit —
  the completed agents' structured results are recoverable and can be synthesized in the main loop.
- **MediaStore over a filesystem walk** for category browsing is the correct, fast primitive on Android
  (indexed by the platform); guard every cursor and degrade to empty on missing permission.
