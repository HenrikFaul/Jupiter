# AI development record — v0.13.0

## Objective

Make every delete recoverable: route deletions through an app-managed Recycle Bin with restore,
permanent-delete, empty, and an audit trail — additively, with an absolute NO-DATA-LOSS guarantee.

## Authoritative user requirement

> "amit már átadtam duplicate handler csomagban (ultimate-duplicate-handler) … még mindig rengeteg
> dolgot tudnál hasznosítani belőle hát hasznosítsd!!!!" — Sift's headline data-safety feature is the
> quarantine/Recycle Bin + restore + safety invariants.

## Implementation constraints

- Additive/no-regression; single-owner-per-file (TR-CORE / TR-DEL / TR-UI, non-overlapping).
- **Cardinal rule**: a file is removed from its original location ONLY once it is safely in trash;
  if it can't be trashed, PRESERVE it (never hard-delete). Trash uses its own non-destructive Room DB.
- CI is the compiler (dl.google.com blocked in-container); "green" only means a successful CI run.
- Duties: changelog + versioning + version bump; capture lessons.

## Lessons captured this iteration

- Directory `deleteRecursively()` can partially succeed then return false — after a verified
  cross-volume copy, retain the target; never destroy the only complete copy on a source-delete miss.
- Route ALL deletes through one authority; hunt for direct `File.delete()` bypasses (there were two).
- A dedicated data-loss verifier in the workflow is high-value for delete-path changes.
  (Sync to `…/lessons-system/lessons_orchestrator.md` from a machine that can reach it.)
