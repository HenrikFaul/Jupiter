A már létező Jupiter Android file manager teljes design- és funkcionális újraauditja, majd a `Generatedscreens/` nyolc képernyőjének hierarchy-, sűrűség-, szín-, tipográfia- és komponensnyelvének ráültetése a valós alkalmazásra. ## Implementation constraints

1. **Nem új app:** a meglévő package, adat-, index-, repository-, route- és feature-rétegre kell építeni.
2. **No regression:** publikus symbol, screen-signature paraméter, enum, direkt elnevezés vagy meglévő route nem távolítható el/írható át. Új paraméter csak a korábbi sorrend után, behavior-preserving defaulttal; új UiState mező a régi viselkedést őrző defaulttal.
3. **Valós állapot:** nincs mock/fiktív tárhely, fájl, keresési snippet, duplikátum, cloud/remote kapcsolat, AI válasz vagy sikerállapot. Inert destructive gomb nem maradhat.
4. **Index invariáns:** Room az autoritatív state; egyetlen rescan; exact csoportot teljes hash dönt el; quick hash csak prefilter; exact és perceptuális similar külön; dHash/pHash/aHash kombinált policy, quality-ranked keeper, generation/sweep és másodperc-pontosság nem sérülhet.
5. **Adatbiztonság:** copy/move self-path és collision tiltás; normál delete csak Recycle Bin; permanent delete explicit megerősítés; retenció default Never; közös `StorageExclusions`; a keeper soha nem kerülhet automatikusan törlésre.
6. **Vault fail-closed:** UI nem állíthat auth-sikert; valós BiometricPrompt/device credential eredmény vagy ellenőrzött PIN szükséges; PIN nem menthető UI-state-be és minden transient másolat nullázandó; storage/Keystore hiba plaintext fallback nélkül zárt állapot; biometria csak PIN mellett kapcsolható ki; external picker sessiont zár, csak adatmentes marker élheti túl az újralétrehozást, az URI memóriabeli és fresh-auth után egyszer használható.
7. **Remote security:** FTPS PROT P, SFTP TOFU host-key rejection, encrypted credential storage és őszinte konfigurációs határok változatlanok.
8. **WorkManager init:** ha az `Application` on-demand `Configuration.Provider` és Hilt `WorkerFactory` konfigurációt ad, a default `androidx.work.WorkManagerInitializer` metaadatot a manifest merge-ben el kell távolítani; más AndroidX Startup initializáló nem törölhető.
9. **Téma:** a midnight/teal a default referencia, de light/system/dynamic user preference működő marad; közös komponens light témában szemantikus surface-et használ, nem fix sötét gradienst.
10. **Nyelv:** valós Android per-app locale API használható; a v0.51 csomag csak angol resource-t tartalmaz, ezért a UI csak System default/English lehetőséget állíthat, fordítást nem szimulálhat.
11. **Evidence honesty:** helyi Gradle, lint, emulator/device és GitHub Actions külön kapuk. Csak a ténylegesen lefutott parancs pontos eredménye írható le; CI-zöldhöz konkrét run id/conclusion kell.
12. **Docs/lessons:** version bump, append-only changelog, summary, AI record és új hibaminta a külső lessons megfelelő részében kötelező.
13. **Git:** `main` branch ellenőrzés → `fetch` → `pull --ff-only origin main` → közvetlen main munka → végső gate → commit → `push origin main`. Dirty user work megőrzendő; reset/checkout tilos.
