# AI development record — v0.10.0

## Objective

Make Dual Pane genuinely useful with **long-press drag-and-drop** copy/move between panes
(plus a Copy/Move drop-mode toggle and phone-appropriate Total-Commander extras), and make
the **duplicate quality comparison** visibly actionable (badge the best copy). Additive,
no regression, no new dependencies.

## Authoritative user requirement

Hungarian, verbatim intent: the dual pane must have a use — at minimum copy, ideally
long-press-and-drag a file to drop it into the other pane's folder, with a toggle bar; add
several dual-pane Total-Commander features that make sense on a phone (e.g. comparison);
show image thumbnails and use space better (thumbnails already shipped in 0.8.0). Also: the
quality scoring/comparison at duplicates was not surfaced — do it.

## Implementation constraints

- No regression; additive/surgical; single-owner-per-file; keep every existing behavior.
- No new dependency (Compose/Coil already present).
- **Compile safety without a local compiler**: `dl.google.com` is blocked in the dev
  container, so CI is the only compiler. To avoid a version-specific compile break, drag was
  implemented with the long-stable `detectDragGesturesAfterLongPress` + `onGloballyPositioned`
  gestures, explicitly NOT the `androidx.compose.foundation.draganddrop` API (whose surface
  differs across Compose 1.6/1.7).
- Honesty: "green" only means a successful CI run id; do not claim compilation otherwise.
- Duties: update `changelog.md` + `versioning/` + bump app version; capture lessons.

## Lessons captured this iteration

- **Compose drag-and-drop API is version-fragile** — for headless/CI-only builds prefer the
  stable `detectDragGesturesAfterLongPress` + `onGloballyPositioned` window-coordinate
  hit-testing over `foundation.draganddrop` to avoid an un-compilable API mismatch.
- **Cross-pane transfer must resolve the destination to the OTHER pane**; set the source VM
  selection before `copySelectedTo/moveSelectedTo`, clear it after, and refresh the
  destination pane (source reloads itself).
- **Surfacing already-computed data**: the quality scores existed since 0.5.0 but weren't
  actionable; badging the max-score copy is the cheap, high-value UX that closes the request.
  (To be synced into `…/lessons-system/lessons_orchestrator.md` from a machine that can reach it.)
